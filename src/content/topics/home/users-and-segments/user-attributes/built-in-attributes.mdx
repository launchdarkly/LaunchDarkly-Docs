## Overview

This topic explains what built-in user attributes are and how to configure them.

LaunchDarkly allows you to target users based on built-in attributes. To learn more, read [Targeting rules](/home/flags/targeting-rules).

## Using built-in user attributes

All built-in user attribute values are `strings` with the exception of `anonymous`, which also accepts a `boolean`.

<Callout intent="info">
<CalloutTitle>LaunchDarkly does not automatically populate built-in attributes</CalloutTitle>
<CalloutDescription>

While LaunchDarkly provides built-in attribute fields for you to fill in, the SDK does not automatically collect any information. If you want to target your users based on any built-in attributes, you must supply their values.<br /><br />

The exceptions to this are the `os` and `device` attributes used by mobile SDKs. Mobile SDKs automatically populate these values.

</CalloutDescription>
</Callout>

The user key is the only mandatory user attribute. All other user attributes are optional.

<Callout intent="info">
<CalloutTitle>Creating searchable user keys</CalloutTitle>
<CalloutDescription>

You can structure your user keys so they are both searchable and unique. To do this, we suggest adding a searchable term to the beginning of the key, followed by a unique ID. For example, if you wanted to easily search users that encountered a specific service, you could structure the key as follows: `service_name:uuid`.

</CalloutDescription>
</Callout>

You can view a user's attributes by clicking on the individual user from the [**Users** list](https://app.launchdarkly.com/users).
